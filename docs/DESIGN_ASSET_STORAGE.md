# DESIGN: 资产存储系统 v1.0

版本: v1.0
日期: 2026-07-21
状态: 📐 design-ready
取代: DESIGN_FILE_FIXES.md, DESIGN_ROOM_V3.md 中的文件管理部分

---

## 核心原则

1. **不同资产，不同规则** — 不是"一个文件管理器适配所有"，是五种资产五种体验
2. **所有权不取决于创建者** — 取决于资产类型和协作规则
3. **并发修改 = 分支，不丢数据** — 两个 AI 同时改一个文件 → 自动 fork
4. **AI 不能碰人上传的资料和自动产出** — 能读、能分析、不能改

---

## 1. 资产分类

| 类型 | 来源 | 谁创建 | 谁拥有 | 能改吗 | 有版本吗 | 生命周期 |
|------|------|--------|--------|--------|---------|---------|
| **产出** | 对话中 AI 生成 + 人修改 | AI / 人 | 房间共有 | AI 和人都能改 | ✅ 版本树 | 持续迭代 → 归档 |
| **资料** | 人从设备上传 | 人 | 上传者 | 只有上传者能删；AI 不能改 | ❌ 覆盖替换 | 长期留存 |
| **归档** | Cron / 技能自动产出 | 系统 | 生成它的任务 | 任何人不能改 | ❌ 每次新文件 | 按保留策略自动清理 |
| **系统** | 房间创建时生成 | 系统 | 系统 | 系统自动更新 | ❌ | 和房间同生命周期 |
| **模板** | 人创建或市场获取 | 人 / 市场 | 创建者（副本） | 所有者能改 | ✅ 版本 | 跨房间复用 |

## 2. 产出 (对话产物) — 版本树

### 磁盘结构

```
/sdcard/mov/rooms/<id>/work/
  Login.tsx                  ← 当前版本 (纯文本, 直接可用)
  .mov/work/
    Login.tsx.json           ← 版本元数据
    Login.tsx/
      v1.tsx                 ← 历史版本快照
      v2.tsx
      v3.tsx
```

### 元数据

```json
{
  "file": "Login.tsx",
  "currentVersion": 4,
  "lock": null,                     // "you" | "DeepSeek" | null
  "versions": [
    {
      "v": 1,
      "author": "DeepSeek",
      "at": 1721404800000,
      "msg": "首次生成 — 来自对话: \"写个登录页面\"",
      "chatLink": "msg_abc123",     // 跳回对话里的那条消息
      "snapshot": "Login.tsx/v1.tsx"
    },
    {
      "v": 2,
      "author": "Claude",
      "at": 1721408400000,
      "msg": "review 修改: 加验证逻辑, 改 40 行",
      "chatLink": "msg_def456",
      "snapshot": "Login.tsx/v2.tsx",
      "parent": 1
    },
    {
      "v": 3,
      "author": "you",
      "at": 1721412000000,
      "msg": "手动改了标题和结构",
      "snapshot": "Login.tsx/v3.tsx",
      "parent": 2
    },
    {
      "v": "4a",
      "author": "DeepSeek",
      "at": 1721415600000,
      "msg": "并行修改标题 — 和 Claude v4b 冲突",
      "snapshot": "Login.tsx/v4a.tsx",
      "parent": 3,
      "branch": true
    },
    {
      "v": "4b",
      "author": "Claude",
      "at": 1721415600000,
      "msg": "并行修改 — 加了新组件",
      "snapshot": "Login.tsx/v4b.tsx",
      "parent": 3,
      "branch": true
    }
  ]
}
```

### 并发规则

```
文件状态:
  lock = null       → 任何人都可以请求修改
  lock = "you"      → 你正在编辑。AI 看到锁 → 生成完存为"待审核"，不覆盖
  lock = "DeepSeek" → DeepSeek 正在生成。人不能改

锁的释放:
  - 人: 关闭编辑器 → 锁释放
  - AI: 生成完成 → 锁释放
  - 超时: 5 分钟无操作 → 自动释放

冲突:
  两个 AI 同时请求写同一个文件 → 后完成的自动分支 (branch: true)
  人想改但 AI 正在生成 → 提示"DeepSeek 正在修改，等它完成？[等] [强制编辑]"
```

### UI

```
┌─ 产出 · src/Login.tsx ────────────────────┐
│                                            │
│  当前: v4a (正在解决冲突)                   │
│  ┌──────────────────────────────────────┐  │
│  │ import { lazy } from 'react'         │  │
│  │ ...                                  │  │  ← 带 diff 高亮的代码视图
│  └──────────────────────────────────────┘  │
│                                            │
│  版本树                                    │
│  v1 ─── v2 ─── v3 ──┬── v4a (当前)        │
│  DS      CL    你    │                     │
│                      ├── v4b               │
│                      │   CL               │
│                      │                     │
│  v4a vs v4b 冲突:                          │
│  标题不同 + 变量名不一致                    │
│  [保留 v4a] [保留 v4b] [并排对比]          │
│                                            │
│  来源                                      │
│  v1: "写个登录页面" → [跳到对话]           │
│  v2: "性能不行, 用懒加载" → [跳到对话]     │
└────────────────────────────────────────────┘
```

## 5. 系统文件 — 隐藏

```
/sdcard/mov/rooms/<id>/.mov/
  config.json           ← 房间配置 (名称/成员/创建时间)
  work/*.json           ← 产出版本元数据
  inbox/index.json      ← 资料元数据
  archive/config.json   ← 归档配置
```

**不在文件 tab 展示。** 用户不需要知道这些。

## 6. 模板 — 跨房间模板库

### 存储

```
/sdcard/mov/templates/
  prd-template.md
  周报模板.md
  code-review-checklist.md
  .mov/templates/
    index.json
```

**模板不属于任何房间。** 所有房间共享。

### 元数据

```json
{
  "prd-template.md": {
    "name": "PRD 模板",
    "desc": "产品需求文档标准模板",
    "author": "you",
    "source": "manual",           // "manual" | "marketplace"
    "version": 2,
    "tags": ["产品", "文档"],
    "usageCount": 5               // 在多少个房间被使用过
  }
}
```

### 规则

| 操作 | 权限 |
|------|------|
| 创建 | 任何人 |
| 使用（复制到房间） | 任何人 |
| 修改模板本身 | 创建者 |
| 删除 | 创建者 |
| AI 改 | ✅ 可以（改的是你自己的副本） |

### 入口

从文件 tab 的 [＋] 按钮:
```
[新建文件]  ← 在当前房间创建
[从模板新建] ← 选模板 → 复制到当前房间
[上传文件]  ← 从设备选
```

## 8. 技术实现

### Java 层不改动

`doFileWrite/Read/Delete/Mkdir` 保持通用。资产逻辑在 JS 层 + 元数据 JSON 文件实现。

### 新增 JS 文件

| 文件 | 职责 |
|------|------|
| `js/assets/work.js` | 产出: 版本树、锁管理、冲突检测 |
| `js/assets/inbox.js` | 资料: 标签、上传、预览 |
| `js/assets/archive.js` | 归档: 来源分组、保留策略 |
| `js/assets/template.js` | 模板: 创建、使用、跨房间 |
| `js/assets/meta.js` | 元数据读写 (通用 JSON 层) |

### 元数据层

所有资产类型共享一个元数据读写抽象:

```javascript
// meta.js
function loadMeta(roomId, assetType) → JSON
function saveMeta(roomId, assetType, data)
function getAssetByType(roomId, type) → {产出: [...], 资料: [...], 归档: [...]}
```

实际文件存在磁盘（Java 能力），元数据存在 `.mov/` 下的 JSON 文件（JS 通过 `B.readFile` / `B.writeFile` 操作）。
