## 用户场景

一个产品经理，打开 MOV，"产品 V2.0"房间。最近三天，他和同事张三、以及 Claude、DeepSeek 一起在这个房间里讨论了登录功能。

AI 生成了好几版代码，他自己改过两版，张三给过意见。现在他想找到两天前 Claude 改的那版 Login.tsx，看看当时 Claude 改了什么，然后决定要不要用那版替代当前的。

**他不是在"管理文件"——他是在追溯决策过程。**

## 3. 操作

### 新建文件

```
＋ → [新建空白文件] → Sheet:
  文件名: [______________]
  内容:   [______________]
  → "创建" → 文件出现在列表

＋ → [从模板新建] → 选模板 → 复制到当前房间 → 改名
```

### AI 写入文件

AI 在对话中说"我生成了代码，要不要存？" → 人点"存" → 文件进入产出列表。元数据自动记录来源对话。

### 重命名

```
长按文件卡片 → "重命名" → Sheet 输入框 → 确认
```

**规则**: 重命名不会影响版本链。版本快照目录也同步改名。

### 删除

```
长按 → "删除" → Sheet 确认 → 文件 + 所有版本快照一起删
规则: 只有房间创建者可以删。
```

### 导出

```
长按 → "导出" → 复制到 /sdcard/Download/
或 → "分享" → Android Share Intent
```

### 复制到别的房间

```
长按 → "复制到房间" → 选目标房间 → 文件的当前版本复制过去。版本历史不带过去。
```

## 4. 版本规则

| 场景 | 行为 |
|------|------|
| AI 写了新版本 | 旧版本 → 快照。新版本 → 当前文件。版本号 +1。 |
| 你手动改了文件 | 同上 |
| 两个 AI 同时改 | 后保存的自动分支 (v4a, v4b)。提示"有并行修改"。 |
| 你正在编辑 | 文件加锁。AI 不覆盖。AI 生成完 → 存为"待审核"。 |
| 你选了一个旧版本 | "应用此版本" → 当前文件被覆盖。旧版本保留在快照里。版本号继续 +1。 |

## 5. 关联到对话

每个版本记录它在哪条消息中产生：

```
v1: 来源 msg_abc123 → "写个登录页面" → 点这行跳到那条消息
v2: 来源 msg_def456 → "性能不行, 用懒加载"
```

反过来，在聊天里看到文件引用卡片：

```
[DS] ⬆ src/Login.tsx
     ┌─────────────────────┐
     │ src/Login.tsx v1     │
     │ DeepSeek · 刚刚      │
     │ [查看] [下载]        │
     └─────────────────────┘
```

消息和文件是双向链接的。谁生成了什么、谁改了什么——在聊天和文件 tab 里都能看到。

## 6. 搜索

```
用户在搜索框输入: "登录"
→ 匹配: 文件名 Login.tsx ✓
→ 匹配: 标签 #登录 ✓  
→ 匹配: 版本描述 "写个登录页面" ✓
→ 不搜: 文件内容 (太慢, 第一版不做)

结果:
  📄 Login.tsx · DeepSeek · 2h 前
  📄 auth-middleware.ts · Claude · 昨天 (标签: #登录 #auth)
```

以后加全文搜索时，在 index.json 里维护一个简单的倒排索引。
