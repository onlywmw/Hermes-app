## 用户场景

产品经理设了两个 Cron 任务："每天早上 8:30 汇总未读邮件"、"每周一检查腾讯云账单"。它们自动运行，产出每天/每周的报告。

三周后，他想回头看 7 月 15 号的邮件汇总——那天有一封重要邮件需要追溯。他不记得那天的文件名，也不记得日期，只记得"大概是 7 月中旬"。

**他不是在浏览文件——他是在翻日历。**

## 3. Cron 产出的特殊性

和项目产出不同——Cron 产出是：

- **只读的。** 它是"报告"，不该被编辑。改了也没意义，下次运行就覆盖了。
- **不需要标题。** 标题 = 来源名 + 日期。自动生成。
- **不需要组织。** 按来源自动分组。用户不需要手动整理。
- **不需要版本。** 每次运行是新文件。不覆盖旧版本。
- **需要过期。** 三个月的邮件汇总没人会看。自动清理。

## 4. 保留策略

### 用户在 Cron 任务设置里配置

```
该任务的产出保留:
  ○ 保留最近 7 天
  ● 保留最近 30 天     ← 默认
  ○ 保留最近 90 天
  ○ 永久保留
  ○ 自定义: 保留最近 [__] 份
```

### 清理行为

- 每天 Cron 运行前检查 → 该来源的归档文件 → 超期的自动删除
- 删除 = 物理删除文件
- 不影响其他来源的归档

### 用户手动清理

```
长按来源分组 → "清空此来源的归档" → Sheet 确认
长按单个归档 → "删除" → Sheet 确认
```

## 5. 失败的归档

Cron 执行失败了——没产出。这一天在列表里怎么显示？

```
┌─── 7月14日 ─────────────────────────┐
│ ⚠️ 执行失败                           │
│ 原因: API Key 过期                    │
│ [重试] [查看 Cron 设置]               │
└─────────────────────────────────────┘
```

- **有失败标记，但没有文件可以点。** 用户看到发生了什么，可以去修 Cron。
- **连续失败提醒。** 同一个 Cron 连续失败 3 次 → 在运行 tab 和房间顶栏提示。

## 6. 操作

| 操作 | 入口 | 行为 | 权限 |
|------|------|------|------|
| 查看 | 点日期 | 打开归档内容 (纯文本预览) | 任何人 |
| 导出 | 长按 / 查看页 [导出] | 复制到 Download | 任何人 |
| 分享 | 查看页 [分享] | Android Share Intent | 任何人 |
| 清空来源 | 长按来源分组 → 清空 | Sheet 确认 → 该来源所有归档删除 | 房间创建者 |
| 删单份 | 长按 → 删除 | Sheet 确认 | 房间创建者 |
| 修改保留策略 | Cron 任务设置页 | 下拉选择 | 房间创建者 |
| 编辑 | — | **不允许** | — |

## 7. 和另外两种资产的区别

| | 产出 (Work) | 资料 (Inbox) | 归档 (Archive) |
|------|-----------|------------|--------------|
| 来源 | AI + 人 | 人上传 | Cron 自动 |
| 改？ | 能 | 不能 | 不能 |
| 版本 | 有 | 无 | 每次新文件 |
| 删 | 房间创建者 | 上传者 + 创建者 | 房间创建者 |
| 展示 | 按时间/作者/来源 | 网格/列表 + 标签 | 按来源 + 日期 |
| 过期 | 无 | 无 | 有保留策略 |
