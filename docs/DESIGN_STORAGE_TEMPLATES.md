# DESIGN: 模板库 — 存储设计 (Templates)

版本: v1.0
日期: 2026-07-22
状态: 📐 design-ready

---

## 用户场景

产品经理做了三个项目房间，每个都要写 PRD。他写了一份"PRD 模板.md"，放在房间 A 里。两周后开房间 B——又得写 PRD。他想起房间 A 里有一个模板，但不记得名字、不记得放哪了。

他需要一个**跨房间的模板库**。"我做过一次的东西，下次用的时候能找到。"

## 2. [使用] 一个模板

```
点击 [使用] →
  ┌─ 从模板新建 ──────────────────────────┐
  │                                        │
  │ 模板: PRD 模板                          │
  │                                        │
  │ 目标房间:                               │
  │ ● 产品 V2.0 (当前房间)                  │
  │ ○ 健身 APP                              │
  │ ○ 新建房间                              │
  │                                        │
  │ 文件名:                                 │
  │ ┌────────────────────────────────┐     │
  │ │ PRD-产品V2.0.md                │     │  ← 自动填入 "模板名-房间名.md"
  │ └────────────────────────────────┘     │
  │                                        │
  │ [创建] [取消]                           │
  └────────────────────────────────────────┘

→ 创建 → 文件出现在目标房间的"产出"列表里。
  模板的 usageCount +1。
```

## 3. 编辑模板本身

```
点击 [编辑] → 打开编辑视图:
  ┌─ 编辑模板: PRD 模板 ────────────────────┐
  │                                          │
  │ 名称: [PRD 模板_________________]        │
  │                                          │
  │ 标签: [产品 文档_______________]          │
  │                                          │
  │ 内容:                                    │
  │ ┌──────────────────────────────────────┐ │
  │ │ # 产品需求文档                        │ │
  │ │                                      │ │
  │ │ ## 概述                              │ │
  │ │ [项目名称] 是一个...                  │ │
  │ │                                      │ │
  │ │ ## 功能列表                          │ │
  │ │ ...                                  │ │
  │ └──────────────────────────────────────┘ │
  │                                          │
  │ [保存] [取消]                            │
  └──────────────────────────────────────────┘

保存 → 模板版本 +1。已使用此模板的房间不受影响（不会自动更新已有文件）。
```

### 编辑规则

| 规则 | 说明 |
|------|------|
| 谁可以编辑 | 创建者。其他人看到模板，不能改。 |
| AI 能改吗 | 能。创建者让 AI 帮忙改模板。AI 可以读模板内容然后修改。 |
| 编辑后会影响已创建的文件吗 | **不。** 模板是种子，种子改了已经长出来的树不会变。 |
| 版本 | 模板有版本。编辑 → 旧版本保留。 |

## 4. 新建模板

```
[新建模板] → Sheet:
  名称: [_______________]
  标签: [_______________] (空格分隔)
  内容: [_______________]

→ 创建 → 模板出现在模板库。初始版本 v1。
```

### 也可以从现有文件创建

```
在"产出"tab 里 → 长按一个文件 → "保存为模板"
→ 文件名自动填入 → 确认 → 模板库多了一项
```

## 5. 操作

| 操作 | 入口 | 行为 | 权限 |
|------|------|------|------|
| 使用模板 | 文件 tab ＋ → 从模板新建 / 模板库 [使用] | 复制到房间 → usageCount +1 | 任何人 |
| 预览 | [预览] | Overlay 显示模板内容 (只读) | 任何人 |
| 新建 | [新建模板] / 长按文件 → 保存为模板 | Sheet | 任何人 |
| 编辑 | [编辑] | 打开编辑视图 | 创建者 |
| 重命名 | ⋮ → 重命名 | Sheet | 创建者 |
| 删除 | ⋮ → 删除 | Sheet 确认 → 模板 + 版本历史删除 | 创建者 |
| 导出 | ⋮ → 导出 | 复制到 Download | 任何人 |

## 6. 和另外四种资产的区别

| | 产出 | 资料 | 归档 | 模板 |
|------|------|------|------|------|
| 在哪 | 房间内 | 房间内 | 房间内 | **跨房间** |
| 来源 | AI + 人 | 人上传 | Cron | 人创建 (或 AI 帮忙) |
| 用途 | 项目文件 | 参考素材 | 自动报告 | **被复制使用** |
| 改？ | 能 | 不能 | 不能 | 能 (仅创建者) |
| 版本 | 版本树 | 无 | 每次新文件 | 版本 |
| 删 | 房间创建者 | 上传者 + 创建者 | 创建者 | 创建者 |
| 使用次数 | — | — | — | 计数: 在多少房间被使用了 |

## 7. 模板和技能市场的关系 (以后)

```
模板 = 免费、自己用的
技能市场 = 付费、别人做的

以后模板可以"发布到技能市场" → 别人下载 → 付费。
版本分叉: 你自己的副本可以继续改; 市场里的版本另走。
```
