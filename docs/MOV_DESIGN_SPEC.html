MOV.多模型协作工作台 · 设计规范

一台平板上的工作台，不是消费 App。人和 AI 在房间里共事，机器在暗处报告状态。 整套语言只有两种材质——纸与终端，一种颜色——信使金。 金色不是装饰，是信号：它只在"活着"的地方出现。

00

设计原则

PRINCIPLES
01

纸与终端

浅色区是"纸"——人写的内容、对话、文件，白底黑字，安静克制。深色区是"终端"——机器的自述：进程、Agent、心跳、日志。两种材质永不混用：聊天里出现深色块，那一定是机器在说话。

--bg / --code-bg
02

字体即身份

等宽字体是系统的声音：时间、状态、路径、徽章、pid 全部 mono。无衬线是人的声音：房间名、消息正文、描述。不需要读内容，看一眼字体就知道这句话是谁说的。

--font-mono / --font-sans
03

层级靠结构，不靠装饰

没有渐变、没有图标库、没有玻璃拟态。层级来自：字号对比（10px 标签 vs 18px 标题）、左侧色条、终端深块、留白节奏。工具的美感来自秩序，不来自点缀。

结构 / 字重 / 留白

金色法则 —— 信使金只属于"活着"的东西

  • 当前选中的 tab / 选项
  • 运行中 · 进行中的状态点
  • 未读标记
  • 主按钮（每屏至多一个）
  • 输入焦点环
  • 进度条填充

禁用：金色标题、金色边框装饰、金色图标、大面积金色背景。静态元素一律灰阶。一屏之内金色出现超过 3 处，说明有东西在假装"活着"。

BEFORE → AFTER
1
BEFORE
运行 tab 结构太散

AFTER

PROC 深色 hero 卡打头；MODEL 折叠进 PROC 尾部；PERMISSIONS/SKILLS 收进设置。每段用 mono 小标签 + 横线 分隔，层级一目了然。
2
BEFORE
"议会/单聊"是技术术语

AFTER
场景化双卡选择
"让 MOV 帮我做"与"拉 AI 团队一起讨论"，各带一句话说明 + 成员头像预览。council/单聊 降级为卡片内的 mono 小标签，仅供识别。

BEFORE

seat-ref.png / PRD-v1.md 永远存在，误导。
AFTER
只渲染真实文件
托盘空时显示 虚线占位"点 + 添加附件"；附件行仅由真实上传/产出驱动，每项可 ✕ 移除。

聊天区底色退到 --bg 纸面，气泡浮起；机器输出（工具卡展开、代码块）进深色终端块。产出物文件卡左金条。三种材质，三种含义。

AFTER

0ms 按压形变 → 500ms 金 outline + tint 底 + 震动 → 操作条浮出。高亮持续到操作完成或取消，状态全程可见。

无图标库 · unicode 排版字符 · 等宽字体渲染
