# DESIGN: 个人内容 — 存储设计 (Personal)

版本: v1.0
日期: 2026-07-22
状态: 📐 design-ready

---

## 用户场景

MOV 不只是协作工具。用户一个人时——看板 tab 打开，听歌、写笔记、看视频、记录健身。这些**不属于任何项目，不共享给同事，不和 AI 协作。** 就是自己的东西。

这和微信的"收藏"、苹果的"备忘录"、Spotify 的"你的音乐库"一样——**个人私有的内容空间。**

---

## 1. 四种个人内容

### 1.1 笔记

用户写的东西。想法、灵感、草稿、日记。

```
数据特征:
  - 纯文本 (Markdown)
  - 单个文件 < 50KB
  - 数量: 几十到几百条
  - 读写: 偶尔写，经常翻看
  - 不需要版本
  - 需要搜索
  - 不需要标签 (标题就是最好的标签)

组织方式:
  按时间倒序。类似备忘录 App。
  支持固定 (pin) 几条到顶部。
  支持搜索标题和内容。
```

### 1.2 音乐

用户存在本地的 MP3/FLAC/M4A，或者流媒体链接。

```
数据特征:
  - 音频文件 (几 MB 到几十 MB)
  - 数量: 几十到几百首
  - 读写: 几乎只读不写
  - 不需要版本
  - 需要按专辑/歌手/歌单组织
  - 需要播放列表

组织方式:
  按歌单 (用户自己建)。
  默认歌单: "所有歌曲" (按添加时间倒序)。
  不自动分析标签——用户自己管歌单。
```

### 1.3 视频

用户存的视频文件，或者视频 URL。

```
数据特征:
  - 视频文件 (几十 MB 到几百 MB)
  - 数量: 几个到几十个
  - 读写: 几乎只读不写
  - 不需要版本
  - 不需要复杂组织——视频少，列表够用

组织方式:
  按添加时间倒序的简单列表。
  支持"稍后观看"标记。
```

### 1.4 健身

训练记录。每次运动后手动记、或者健身 App 自动记录。

```
数据特征:
  - 结构化数据 (日期、动作、组数、次数、时长)
  - 每条 < 200B
  - 数量: 每天一条，一年 365 条
  - 读写: 每天写一次，偶尔回看
  - 不需要版本
  - 需要按周/月汇总

组织方式:
  日历视图。类似运动 App 的月历。
  点日期 → 看当天的训练详情。
  周汇总: 本周运动 X 次，总时长 Y 分钟。
```

## 2. 统一存储策略

四种个人内容共享一个存储层，但有各自独立的子目录和元数据：

```
/sdcard/mov/personal/
  notes/
    2026-07-22-产品想法.md
    2026-07-20-会议记录.md
    2026-07-15-灵感碎片.md
  music/
    歌单A.json           ← 歌单 = 文件路径列表
    歌单B.json
    tracks/               ← 实际音频 (或用 /sdcard/music/ 共享目录)
  videos/
    产品演示.mp4
    教程-React进阶.mp4
    watchlist.json        ← 稍后观看列表
  fitness/
    2026-07.json          ← 每月一个文件, 每天一条记录
    2026-06.json
  .meta/
    notes.json            ← 笔记列表 + 元数据
    music.json            ← 歌单清单 + 曲目列表
    videos.json           ← 视频列表
    fitness.json          ← 训练汇总统计
```

### 写笔记

```
点击 [+] → 全屏编辑器:
  ┌─ 新建笔记 ────────────────────────────┐
  │ [完成]                         [⋮]   │
  │                                       │
  │ 标题                                  │
  │ ┌───────────────────────────────────┐ │
  │ │ 功能优先级矩阵                     │ │
  │ └───────────────────────────────────┘ │
  │                                       │
  │ ┌───────────────────────────────────┐ │
  │ │ ## P0 - 必须做                    │ │
  │ │ - 用户注册登录                    │ │
  │ │ - 商品浏览搜索                    │ │
  │ │                                   │ │
  │ │ ## P1 - 应该做                    │ │
  │ │ - 购物车                          │ │
  │ │ - 个人中心                        │ │
  │ │ ...                               │ │
  │ └───────────────────────────────────┘ │
  └───────────────────────────────────────┘

自动保存: 停止输入 2 秒后自动保存。
标题: 不填 → 自动取第一行。
```

### 操作

| 操作 | 行为 |
|------|------|
| 新建 | ＋ → 全屏编辑器 |
| 编辑 | 点笔记 → 全屏编辑器。改了自动保存。 |
| 固定 | 长按 → 固定到顶部 |
| 删除 | 长按 → 删除 |
| 搜索 | 搜标题 + 内容 |
| 导出 | 长按 → 导出 .md 到 Download |
| 发给房间 | 长按 → "发送到房间" → 选房间 → 内容复制到聊天消息 |

### "发给房间"是什么

用户在笔记里写了一个产品方案，觉得可以拿到项目房间里讨论 → 点"发送到房间" → 笔记内容变成一条聊天消息发出去。不是文件——是消息。对方（或 AI）可以在聊天里继续讨论。

**笔记可以升级为协作产出，但不自动。** 用户主动触发。

```
    "/sdcard/music/lo-fi-beat-3.mp3",
    "/sdcard/music/ambient-waves.mp3"
  ]
}
```

## 7. 个人内容和协作的边界

| 操作 | 行为 |
|------|------|
| 个人 → 房间 | 笔记可"发送到房间"变成聊天消息。健身/音乐/视频不发给房间。 |
| 房间 → 个人 | 房间里的文件可"保存到笔记"。AI 的回复可"记下来"。 |
| 备份 | 整个 `/sdcard/mov/personal/` 目录可导出 ZIP。 |
| 删应用 | 删健身应用 → 训练数据保留。删 MOV → personal 目录还在（在 /sdcard/mov/ 下，不随 APK 卸载删除）。 |

## 8. 五种存储设计的完整对比

| | 产出 | 资料 | 归档 | 模板 | 个人 |
|------|------|------|------|------|------|
| **范围** | 房间内 | 房间内 | 房间内 | 跨房间 | 设备级 |
| **来源** | AI + 人 | 人上传 | Cron | 人 (或 AI帮) | 人 |
| **共享？** | 房间成员 | 房间成员 | 房间成员 | 所有房间 | **仅自己** |
| **改？** | 能 | 不能 | 不能 | 能(创建者) | 能 |
| **版本** | 版本树 | 无 | 每次新文件 | 版本 | 无 |
| **删** | 创建者 | 上传者+创建者 | 创建者 | 创建者 | 自己 |
| **过期** | 无 | 无 | 保留策略 | 无 | 无 |
| **搜索** | 文件名+标签+版本描述 | 文件名+标签+AI摘要 | 来源名+日期 | 名称+标签 | 标题+内容 |
| **组织** | 时间/作者/来源 | 网格/列表+标签 | 来源+日期 | 分类标签 | 时间倒序 |
