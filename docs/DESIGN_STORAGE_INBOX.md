## 用户场景

产品经理在"产品 V2.0"房间里。甲方发了三份 PDF 需求文档，张三上传了两张竞品截图，他自己也传了一份 Excel 数据表。这些不是 AI 生成的——它们是项目的 **参考素材**。放在这里供人和 AI 随时查阅。

一周后，他想找"甲方发的那份关于支付流程的 PDF"。他不记得文件名，只记得是甲方发的、关于支付的。

**他不是在翻文件夹——他是在翻记忆。**

## 3. AI 分析

### 触发方式

```
资料上传后 → 自动触发 (可选, 设置里开关)
或: 长按资料 → "AI 分析"
```

### 分析结果

```
AI 读了这个文件，生成一段摘要:

  对于图片:
  "这是一个电商 APP 首页设计稿。布局从上到下: 搜索栏、轮播 Banner、
  分类图标、商品推荐网格。整体白色背景, 使用橙色作为主色调。"

  对于 PDF:
  "这是一份产品需求文档。主要内容包括: 用户注册登录、商品浏览搜索、
  购物车、支付流程、订单管理。其中支付部分要求支持微信和支付宝。"
```

### 摘要怎么被使用

1. 在资料卡片上显示 (折叠/展开)
2. 在聊天里 AI 可以直接引用: "根据你上传的竞品截图, 他们的首页有..."
3. 搜索时匹配摘要文字: 搜"支付" → 找到包含支付内容的 PDF

## 4. 标签系统

### 添加标签

```
上传时: 可选填标签
上传后: 长按 → "编辑标签" → 输入框, 支持多个标签 (空格分隔)
AI 自动建议: 资料上传后, AI 分析完 → 自动推荐 3 个标签 → 用户确认或修改
```

### 标签的作用

- **筛选**: 点击顶部的 `#甲方` → 只看带这个标签的资料
- **搜索**: 标签文字可被搜索
- **跨房间**: 标签是房间级的。不同房间的资料标签独立

### 预置标签 vs 自定义

不预置。用户自己创建。常见的标签会自然浮现（#甲方 #竞品 #需求 #参考 #设计稿）。以后可以基于使用频率推荐标签。

## 5. 操作

| 操作 | 入口 | 行为 | 权限 |
|------|------|------|------|
| 上传 | 右上角 [上传] / 文件 tab ＋ | 系统文件选择器 | 任何人 |
| 预览 | 点击资料 | 图片全屏 / PDF 阅读器 / 文本视图 | 任何人 |
| 重命名 | 长按 → 重命名 | Sheet 输入框 | 任何人 |
| 加标签 | 长按 → 编辑标签 | 输入标签 | 任何人 |
| 导出 | 长按 → 导出 | 复制到 Download 目录 | 任何人 |
| AI 分析 | 自动或长按 → AI 分析 | AI 读取文件生成摘要 | 任何人 |
| 删除 | 长按 → 删除 | Sheet 确认 | **上传者** + 房间创建者 |

## 7. 和产出的区别

| | 产出 (Work) | 资料 (Inbox) |
|------|-----------|------------|
| 来源 | AI 生成 + 人修改 | 人从设备上传 |
| 能改吗 | AI 和人都能改 | AI 不能改, 人不能直接编辑 |
| 版本 | 版本树 | 无版本, 覆盖替换 |
| 归属 | 房间共有 | 上传者的 |
| 删除 | 房间创建者 | 上传者 + 创建者 |
| 搜索 | 文件名 + 标签 + 版本描述 | 文件名 + 标签 + AI 摘要 |
| 展示 | 列表, 按时间/作者/来源 | 网格 (图片) 或列表 (文档) |
