AI 动手前先交计划，批准即授权路径；每一次写盘前强制弹出内容预览卡，你点「确认写入」才落盘，驳回则打回重做。

Plan Gate · Preview Card
CAP·03

30 个设备能力

「把手电筒打开」「音量调到 30」——指令优先路由，命中即原生执行，未命中才交给 AI。手电、亮度、剪贴板、TTS、通知、进程列表……

IntentParser → CapabilityExecutor
CAP·04

版本快照

人和 AI 的每次覆盖写入都先自动快照。改坏了？文件卡片上一键回到任意历史版本，work-snapshots 目录替你记着一切。

snapshotWorkFile · restore
CAP·05

Cron 白名单定时

「每天早上 9 点播报电量」——WorkManager 调度，创建时校验 + 运行时兜底双闸，白名单外的动作永远排不上日程。

CronPolicy · 双闸校验
CAP·06

HTML → 签名 APK

AI 在房间里写了个贪吃蛇？一条 app.package 指令把它打包成签名 APK 直接装手机。从对话到应用，不出这个房间。

app.package · PackageBuilder
03

红线写进架构，
不靠自觉

CONTRACT_SECURITY.md — 违反即打回，没有例外。

RULE·01 确认才落盘 所有 AI 文件写入必须经过预览卡片，用户确认前一个字节都不会写进存储。
RULE·02 Cron 白名单 定时任务只能执行 CronPolicy 白名单内的 19 类本地动作，创建与运行双重校验。
RULE·03 数据不出手机 存储根目录锁定 getExternalFilesDir，模型 Key 走 EncryptedSharedPreferences 加密落盘。
RULE·04 渲染零注入 DOM 一律 createElement + textContent，富文本过白名单 sanitize，AI 输出当不可信输入处理。

05

五分钟，
从源码到手机

Java 11 · Android API 26+ · targetSdk 36 · Gradle 8.13

  1. 克隆仓库一个 Git 命令，MIT 协议随便用。
  2. 构建 APKGradle 一把梭，无前端框架、无原生依赖地狱。
  3. 装进手机adb 直装，Android 8.0 以上即可。
  4. 接入模型运行页填 Key，建房间，下第一个目标。
bash — mov
# 克隆 $ git clone git@github.com:onlywmw/MOV-APP.git # 构建 $ ./gradlew assembleDebug BUILD SUCCESSFUL # 安装 & 启动 $ adb install -r app/build/outputs/apk/debug/app-debug.apk $ adb shell am start -n com.hermes.android/.HermesActivity # 跑测试 · 158 个用例 $ ./gradlew test
